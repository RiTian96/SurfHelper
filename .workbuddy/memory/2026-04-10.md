## 优化后的开发工作流程

基于本次 JavDB Manager v2.2.0 开发经验，确立以下工作流：

### 迭代开发阶段（可多次循环）
1. **需求提出** — 用户提出功能需求或优化点
2. **开发实现** — 修改 `.user.js` 脚本代码
3. **功能验证** — 用户在实际环境测试功能
4. **Git 保存** — 验证通过后，分步提交到本地 Git
   - 每个独立功能/优化单独提交
   - 便于出错时精准回退

### 最终发布阶段（开发结束后执行）
5. **版本号更新** — 更新脚本 `@version`
6. **文档同步** — 更新对应 `.md` 文档、CODEBUDDY.md、README.md
7. **最终确认** — 用户确认所有改动无误
8. **推送发布** — 推送到 GitHub

### 关键原则
- **开发过程中不更新版本号和文档** — 避免碎片化提交
- **每个功能验证后立即 Git 提交** — 确保可回退
- **分步提交优于单个大提交** — 精准定位问题
- **最终统一更新文档** — 保持信息一致性
